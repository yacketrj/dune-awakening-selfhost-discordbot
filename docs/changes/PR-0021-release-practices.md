# PR-0021: Add Release Practices

## Summary

Adds a security-first release process for the private Discord bot repository.
The change introduces release metadata validation, unit tests for the release
validator, first-release notes, a changelog, and a tag-driven GitHub Release
workflow that publishes checksummed addon and SBOM artifacts.

## User Impact

Operators get a documented release path and a first release note for `v0.1.0`.
Release artifacts remain generated by the repository workflow and include
checksum files for verification.

## Security Impact

- Command surface: unchanged.
- RBAC or authorization: unchanged.
- Secret handling: unchanged; release docs explicitly require redaction for
  secrets, emails, SteamIDs, FuncomIDs, real names, private addresses, and `.env`
  contents.
- Data crossing Discord/bot/WebUI boundaries: unchanged.
- Network exposure: unchanged.
- Release workflow permission: `contents: write` is added only to the release
  artifact workflow so validated tag pushes can create GitHub Releases and
  upload release assets.

## Least Privilege

Runtime least privilege is unchanged. The bot still needs only its Discord bot
token, configured Discord application IDs, read-only WebUI adapter access, and
the existing restricted container privileges. The optional addon remains
zero-permission.

## Tests and Evidence

- `npm run check`
- `npm audit --audit-level=moderate`
- Semgrep
- Gitleaks
- Trivy filesystem
- Dependency review
- SBOM generation
- Docker build
- Trivy image

Local finding and resolution: Semgrep initially flagged dynamic `RegExp`
construction in `scripts/validate-release.js` as a possible ReDoS risk. The
validator was changed to use deterministic changelog line matching, and Semgrep
was rerun with 0 findings. No GitHub issue was opened because the finding was
resolved before the pull request.

## Known Limitations

This PR prepares release mechanics and documentation. The actual `v0.1.0` tag
must be created only after the PR merges to `main` and final gates pass from the
clean `main` branch.
