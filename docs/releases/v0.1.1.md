# Release v0.1.1

Release date: 2026-06-28

## Summary

Stable patch release that promotes the verified `v0.1.1-rc.1` release
candidate. This release adds release-candidate workflow support and updates the
roadmap for RC-to-stable release discipline.

## Compatibility

- Bot repository: `yacketrj/dune-awakening-selfhost-discordbot`
- Upstream stable baseline:
  `Red-Blink/dune-awakening-selfhost-docker@1bb72c5`
- Upstream stable tag reviewed: `v1.3.37`
- Upstream release-candidate tag observed: `v1.3.38-rc.1` at `233aedf`
- Adapter boundary: read-only routes only

No Discord adapter route files changed between upstream `v1.3.37` and
`v1.3.38-rc.1`, so this bot keeps `v1.3.37` as the stable compatibility
baseline.

## Changes Since v0.1.0

- Release validation accepts prerelease SemVer versions such as `0.1.1-rc.1`.
- Addon packaging accepts prerelease SemVer versions and produces matching
  prerelease artifact names.
- The release workflow marks prerelease version tags as GitHub prereleases.
- The roadmap records RC-to-stable release discipline.
- The non-read-only roadmap requires write-capable releases to start as release
  candidates before stable promotion.
- Upstream evidence records the standalone Windows reference clone and the
  latest observed upstream RC separately from the stable baseline.

## Candidate Evidence

`v0.1.1-rc.1` was published as a GitHub prerelease before this stable
promotion. Its release workflow passed and its downloaded addon and SBOM
artifacts were verified with their SHA-256 checksum files.

## Security Posture

- Runtime command behavior is unchanged.
- RBAC behavior is unchanged.
- The bot remains read-only.
- The optional addon remains zero-permission.
- Release notes and PR evidence continue to require redaction of secrets, PII,
  SteamIDs, FuncomIDs, private addresses, and real names.
- No unresolved medium, high, or critical security findings were identified
  during candidate validation.

## Release Artifacts

Verify checksum files before installing or redistributing artifacts.

## Verification

Stable releases must pass:

- `npm run check`
- `npm audit --audit-level=moderate`
- Semgrep
- Gitleaks history and worktree scans
- Trivy filesystem scan
- Docker image build
- Trivy image scan
- published artifact checksum verification

## Known Limitations

- No live Discord guild or live WebUI deployment test is bundled with the
  release; those remain operator smoke tests.
- No write-capable commands are included.
