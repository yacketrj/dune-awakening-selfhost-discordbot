# Release v0.1.0

Release date: 2026-06-28

## Summary

Initial read-only release of the Dune Awakening Self-Host Discord Bot. This
release is intended for operators who run their own Discord application and
connect it to their own Dune WebUI Discord adapter.

## Compatibility

- Bot repository: `yacketrj/dune-awakening-selfhost-discordbot`
- Upstream source of truth:
  `Red-Blink/dune-awakening-selfhost-docker@1bb72c5`
- Upstream tag reviewed: `v1.3.37`
- Adapter boundary: read-only routes only

## Included Commands

- `/dune about`
- `/dune ping`
- `/dune health`
- `/dune status`
- `/dune status-summary`
- `/dune readiness`
- `/dune services`

## Security Posture

- RBAC is restricted by default and fails closed without configured principals.
- The bot does not mount the Docker socket, connect to the database, read game
  files, execute shell commands, or call write-capable adapter routes.
- The optional addon package has zero permissions and is a setup panel only.
- Logs and Discord output use shared redaction for credentials, emails,
  SteamIDs, FuncomIDs, and explicit real-name fields.
- PCI/payment-card data is not expected. If it appears in adapter output, treat
  it as a security finding and stop that data flow.

Release candidates must pass:

- `npm run check`
- `npm audit --audit-level=moderate`
- Semgrep
- Gitleaks history and worktree scans
- Trivy filesystem scan
- Docker image build
- Trivy image scan

The read-only completion review recorded no unresolved medium, high, or critical
security findings.

## Known Limitations

- No hosted shared bot is provided.
- No live Discord guild or live WebUI deployment test is bundled with the
  release; those remain operator smoke tests.
- No write-capable commands are included. Future write-capable work must follow
  `docs/non-readonly-roadmap.md`.
